# Charger Monitor — macOS readiness и инструкция запуска

Дата обновления: 2026-04-03

Кратко
- Этот проект ориентирован на macOS: сбор данных реализован через встроенные системные утилиты macOS (system_profiler и ioreg). Фронтенд — статический HTML (charger_dashboard.html), сервер и сборщик данных — в одном скрипте (charger_server.sh) на bash + встроенном Python3.
- Если вы используете Mac (ноутбук с батареей), проект готов к локальному запуску «как есть». На машинах без батареи (iMac, Mac mini десктоп) данные по батарее будут отсутствовать/нулевые.

Готовность для macOS — оценка
- Поддерживаемые платформы: macOS (Intel и Apple Silicon). Код использует стандартные утилиты system_profiler и ioreg, доступные в macOS независимо от архитектуры.
- Требования:
  - macOS с поддержкой system_profiler и ioreg (все стандартные релизы macOS).
  - Python 3 доступен как python3 в PATH.
  - Bash (входит в macOS).
  - Веб-браузер для просмотра dashboard (http://localhost:7823).
- Ограничения/замечания:
  - Сбор данных ориентирован на ноутбуки Apple (AppleSmartBattery). На десктопах (без батареи) ioreg не вернёт ожидаемых полей.
  - Парсинг значений простой и эмпирический — возможны шумы/погрешности в показаниях (особенно для тока/напряжения). Математика проста: voltage * amperage ≈ power.
  - Скрипт не требует sudo в обычном случае. Если какие-то команды возвращают пустые данные, проверьте доступность утилит и отсутствие ограничений безопасности в системе (редко встречается).

Как запустить (быстро)
1. Откройте терминал и перейдите в каталог, где лежат файлы (скрипт и HTML). В этом репозитории они находятся в папке `backups/`.
   cd path/to/repo/backups

2. Сделайте скрипт исполняемым (один раз):
   chmod +x charger_server.sh

3. Запустите скрипт:
   ./charger_server.sh

4. Откройте в браузере:
   http://localhost:7823

Что делает скрипт
- collect_data (встроенный Python) каждые ~2 секунды вызывает system_profiler SPPowerDataType и ioreg -rn AppleSmartBattery, парсит ключевые поля (Full Charge Capacity, State of Charge, Cycle Count, Voltage, Amperage, Max/Design capacity, Temperature, TimeRemaining, Wattage адаптера) и сохраняет JSON в /tmp/charger_data.json.
- В фоновом цикле данные обновляются каждые 2s.
- Одновременно запускается простой HTTP-сервер (на localhost:7823), который отдаёт charger_dashboard.html и /data.json (файл /tmp/charger_data.json).
- Фронтенд опрашивает /data.json каждые 2s, рендерит график мощности и метрики, позволяет записывать «сессии» в localStorage.

Проверка и отладка
- Если скрипт завершается с ошибкой парсинга, он выдаст содержимое временного JSON. Проверьте вывод команд вручную:
  system_profiler SPPowerDataType
  ioreg -rn AppleSmartBattery
  (в терминале) — убедитесь, что данные есть.

- Python3 не установлен: если команда python3 не найдена, установите Python 3 (например, через brew: brew install python) или назначьте python3 на вашу установленную версию (не рекомендуется менять системный python).

- Порт занят: по умолчанию скрипт использует порт 7823. Если порт занят, остановите процесс, занимающий порт, или измените переменную PORT в начале charger_server.sh и перезапустите.
  Проверка занятого порта (пример):
    lsof -iTCP:7823 -sTCP:LISTEN

- Нет данных от ioreg/system_profiler: убедитесь, что вы запускаете на MacBook с батареей. Для диагностики выполните в терминале:
    system_profiler SPPowerDataType | sed -n '1,200p'
    ioreg -rn AppleSmartBattery | head -n 80

Запуск на удалённой машине / без macOS
- На не-macOS скрипт не соберёт реальные показания батареи. Чтобы протестировать интерфейс, можно создать статический data.json и запустить http.server:
  cp charger_dashboard.html index.html
  echo '{"timestamp":"now","power_w":0,"percent":50}' > data.json
  python3 -m http.server 7823
  # Откройте http://localhost:7823/index.html (dashboard должен запрашивать /data.json)

Безопасность и сеть
- Сервер слушает только localhost (127.0.0.1), поэтому он доступен только локально. Это удобно и безопасно для личного использования.
- Скрипт не включает аутентификацию — использовать только в доверенной сети/на локальной машине.

Рекомендации для улучшения macOS-готовности
- Более устойчивый парсер: добавить защиту от отсутствующих полей, логирование и метрики ошибок.
- Проверки архитектуры/системной версии (если необходимо настроить поведение для старых версий macOS).
- Возможность выбора альтернативных источников данных (например, внешние USB-PD измерители) для тестирования на десктопах.
- Экспорт/импорт сессий (CSV/JSON) вместо хранения только в localStorage.

Короткая инструкция для разработчика (быстрое тестирование)
- Запуск: (в папке backups)
  chmod +x charger_server.sh
  ./charger_server.sh
- Откройте: http://localhost:7823
- Остановить: Ctrl+C в терминале, который запустил скрипт

Если нужно — могу:
- Вставить больше диагностических логов в charger_server.sh (например, включить debug-режим вывода system_profiler и ioreg).
- Подготовить небольшую инструкцию для сборки .app (если нужно упаковать как macOS приложением).
