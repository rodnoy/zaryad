# Plan: Charger Monitor — Native macOS App (xcodeproj + SwiftUI + IOKit + SwiftData)

## Текущее состояние

Проект — macOS-приложение мониторинга батареи. Есть:
- Swift Package Manager структура (Package.swift, без .xcodeproj)
- Clean Architecture: Domain / Data / Presentation / App (4 модуля)
- `ShellSystemPowerRepository` — читает данные через shell-процессы (`ioreg`, `system_profiler`)
- `SwiftDataSessionStore` — **заглушка**, делегирует в `FileSessionStore` (JSON файл)
- SwiftUI views — **базовые**, далеки от веб-дашборда (нет графика с градиентами, нет battery bar, нет metrics grid, нет info grid, нет таблицы сессий с рейтингами)
- Сессии хранятся в JSON файле, исторические сэмплы **не сохраняются**

## Целевое состояние

1. `.xcodeproj` — полноценный Xcode проект для macOS App
2. **IOKit** — нативное чтение батареи без shell-процессов
3. **SwiftData** — персистентное хранение сессий и всех измерений батареи
4. **Полный SwiftUI дашборд** — визуально равный веб-версии, без заглушек

## Фаза 1: Xcode Project

### Задача 1.1 — Создать .xcodeproj

Создать Xcode проект через `xcodegen` или вручную (project.pbxproj). Рекомендуемый подход — **один App target** со всеми исходниками в группах (Domain/, Data/, Presentation/, App/), потому что:
- Упрощает сборку и подпись
- Нет необходимости в отдельных framework-таргетах для solo-проекта
- Можно сохранить папочную структуру как логические группы

**Конкретные шаги:**
1. Создать `ChargerMonitor.xcodeproj` (macOS App, Swift, SwiftUI lifecycle)
2. Target: `ChargerMonitor`, macOS 14.0+ (для SwiftData, `@Observable`)
3. Добавить все .swift файлы из `Sources/` в единый target
4. Удалить `DataLayer` namespace enum и `Presentation` namespace enum — они нужны для мульти-модульного SPM, но мешают в одном таргете. Вместо этого использовать папки как логическую организацию.
5. Создать `Info.plist` с:
   - Bundle name: Charger Monitor
   - Bundle identifier: com.worldproject.chargermonitor
   - LSUIElement: false (обычное оконное приложение)
6. Создать `ChargerMonitor.entitlements`:
   - `com.apple.security.app-sandbox`: false (нужен доступ к IOKit)
   - Или включить sandbox + `com.apple.security.temporary-exception.iokit-user-client-class` для IOKit

**Результат:** Проект открывается в Xcode, билдится в .app

## Фаза 2: IOKit Battery Repository

### Задача 2.1 — Создать `IOKitBatteryRepository`

Заменить `ShellSystemPowerRepository` на нативную реализацию через IOKit.

**Файл:** `Sources/Data/SystemPower/IOKitBatteryRepository.swift`

**Реализация:**
```swift
import Foundation
import IOKit.ps // IOPowerSources
// Альтернатива: IOServiceGetMatchingService для AppleSmartBattery

final class IOKitBatteryRepository: SystemPowerRepository {
    func fetchCurrentSample() async throws -> BatterySample {
        // Способ 1: IOServiceGetMatchingService
        let service = IOServiceGetMatchingService(
            kIOMainPortDefault, // или kIOMasterPortDefault для macOS < 12
            IOServiceMatching("AppleSmartBattery")
        )
        guard service != IO_OBJECT_NULL else {
            throw BatteryError.serviceNotFound
        }
        defer { IOObjectRelease(service) }
        
        // Получить все свойства как CFDictionary
        var props: Unmanaged<CFMutableDictionary>?
        let kr = IORegistryEntryCreateCFProperties(service, &props, kCFAllocatorDefault, 0)
        guard kr == KERN_SUCCESS, let dict = props?.takeRetainedValue() as? [String: Any] else {
            throw BatteryError.propertiesUnavailable
        }
        
        // Парсить поля:
        // "Voltage" (mV) → / 1000 = V
        // "Amperage" (mA, signed) → / 1000 = A  
        // "CurrentCapacity" (%) 
        // "MaxCapacity" (mAh)
        // "DesignCapacity" (mAh)
        // "CycleCount"
        // "Temperature" (centi-°C) → / 100 = °C
        // "IsCharging" (Bool)
        // "ExternalConnected" (Bool)
        // "FullyCharged" (Bool)
        // "TimeRemaining" (min)
        // "AdapterInfo" → Watts (может быть в отдельном сервисе)
        
        // Способ 2 (для AdapterWatts): IOPSCopyPowerSourcesInfo() / IOPSCopyPowerSourcesList()
        // для получения данных адаптера
    }
}
```

**Ключевые моменты:**
- `IOServiceGetMatchingService` + `IORegistryEntryCreateCFProperties` — основной путь для всех полей кроме adapter_watts
- Для `adapterWatts`: использовать `IOPSCopyPowerSourcesInfo()` из `IOKit.ps` или найти сервис `AppleSmartBatteryManager` / `AppleACAdapter`
- Конвертация единиц: Voltage mV→V (/1000), Amperage mA→A (/1000), Temperature centi-°C→°C (/100)
- `powerW` = voltageV * amperageA
- `healthPct` = maxMah * 100 / designMah (вычислять на уровне Domain/View, не здесь)
- Линковать `IOKit.framework` в Xcode target

## Принятые решения

- **macOS 14+ (Sonoma)** — используем SwiftData, `@Observable`, новые API
- **Без App Sandbox** — полный доступ к IOKit, подходит для personal-use app
- Entitlements: `com.apple.security.app-sandbox` = NO

## Критически важные моменты

1. **Шрифты** — JetBrains Mono и Space Grotesk нужно включить в bundle или использовать system monospaced/rounded как fallback.
2. **Один target** — переход от 4-х SPM модулей к одному Xcode target убирает `import Domain/Data/Presentation` и namespace enums, но сохраняет папочную организацию.
3. **Session.name** — в веб-версии сессии имеют имя зарядника. В Swift `Session` модели нет поля `name`. Нужно добавить `name: String?` в `Session`.
